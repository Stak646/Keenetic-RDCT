# Config Migrations

Пошаговые миграции config.json между версиями.

## Формат
- `v0_to_v1.sh` — миграция с версии 0 (без config_version) на версию 1
- Каждый файл экспортирует `migrate_config(file)` функцию
- Перед миграцией создаётся backup: `config.json.pre-migrate.<timestamp>`

## Правила
- Только добавление optional полей с дефолтами (без удаления)
- Тесты на реальных config файлах из предыдущих версий
- После миграции — обязательная валидация по текущей схеме

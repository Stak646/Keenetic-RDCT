# Приватность и редактирование

RDCT собирает диагностику с роутера, поэтому отчёты могут содержать чувствительные данные.

## USB-only

RDCT откажется работать, если базовая папка не находится на внешнем USB.

## Уровни редактирования

Редакция применяется по возможности **во время сбора** и обязательно при **экспорте**.

- `strict` — маскирует секреты/куки/токены; бинарные/большие/чувствительные файлы заменяет stub'ами
- `normal` — маскирует типовые секреты; сохраняет больше текста
- `off` — без редактирования (НЕ рекомендуется для отправки)

## Safe view

WebUI рассчитан на локальное использование. Перед отправкой отчёта всегда делай **Export** со строгой редакцией.

## Что RDCT НЕ делает

- Не отправляет креды (нет попыток логина)
- По умолчанию не делает внешних запросов (`external_traffic_allowed=false`)
- Не запускает сторонние сервисы автоматически

## Рекомендации

- Для быстрого анализа — `light`/`medium`
- Для саппорта — `export --level strict`
